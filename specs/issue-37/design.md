# Issue #37: ダッシュボードのスマホ表示時のデザイン修正

## 問題分析

### 現状の動作

ダッシュボード画面（`/`）では月別サマリーテーブルが表示されるが、スマートフォン（幅 768px 未満）での表示時に以下の問題がある：

1. **テーブルが横に長すぎる**: 13 列（月、件数、男性、女性、会場費、見込み収入、決済済み、未回収、見込み利益、実現利益、見込み利益(CB込)、実現利益(CB込)、利益率）が水平に並び、横スクロールが必要
2. **セル幅が狭く数値が見づらい**: `whitespace-nowrap` で改行禁止だが、狭い画面では視認性が低い
3. **ヘッダーとタイトルの配置**: 年セレクターとタイトルが横並び（`flex items-center justify-between`）で、スマホでは窮屈

### 問題の原因（コード箇所）

| ファイル | 行 | 問題点 |
|---------|-----|--------|
| `src/components/dashboard/monthly-summary-table.tsx` | L30-103 | 13列すべてを常に表示。レスポンシブ対応なし |
| `src/components/dashboard/monthly-summary-table.tsx` | L112 | `overflow-x-auto` で横スクロール可能だが、UX として不十分 |
| `src/components/dashboard/monthly-summary-table.tsx` | L118, L134 | `whitespace-nowrap` でヘッダー・セルが改行禁止 |
| `src/app/page.tsx` | L21-24 | ヘッダー部分のレスポンシブ対応なし |
| `src/components/ui/table.tsx` | L73, L86 | 基底コンポーネントでも `whitespace-nowrap` が設定済み |

## 修正方針

### 採用案: カード形式のレスポンシブデザイン

スマホ表示時（768px 未満）はテーブル形式からカード形式に切り替える。

#### 変更対象ファイルと変更内容

| ファイル | 変更内容 |
|---------|---------|
| `src/components/dashboard/monthly-summary-table.tsx` | スマホ用カードビューを追加。`useMediaQuery` または CSS で表示切替 |
| `src/app/page.tsx` | ヘッダー部分をレスポンシブ対応（縦並びオプション） |
| `src/app/globals.css` | 必要に応じてカスタムスタイルを追加 |

#### 具体的な修正内容

**1. MonthlySummaryTable コンポーネントの修正**

```tsx
// スマホ用カードビュー（768px未満で表示）
<div className="md:hidden space-y-4">
  {rows.map((row) => (
    <MonthCard key={row.month} year={year} row={row} />
  ))}
</div>

// PC用テーブルビュー（768px以上で表示）
<div className="hidden md:block rounded-md border overflow-x-auto">
  <Table>...</Table>
</div>
```

**2. MonthCard コンポーネントの新規作成**

各月のデータをカード形式で表示：
- ヘッダー: 月リンク、件数
- ボディ: 参加者数（男/女）、金額情報をグリッド表示
- 重要な数値（見込み利益、実現利益）を強調

**3. ヘッダー部分のレスポンシブ対応**

```tsx
<div className="flex flex-col gap-2 sm:flex-row sm:items-center sm:justify-between">
  <h1 className="text-2xl font-bold">ダッシュボード</h1>
  <YearSelector currentYear={year} />
</div>
```

### 選択しなかった代替案

| 代替案 | 不採用理由 |
|-------|-----------|
| 列の非表示（スマホで一部列を隠す） | 情報が欠落し、ユーザーが必要なデータを見れない |
| 横スクロールのままUX改善のみ | 根本的解決にならない。スクロール操作が煩雑 |
| テーブルの転置（行列入替） | 12行×13列が13行×12列になるだけで改善効果薄い |
| アコーディオン形式 | 詳細表示に追加タップが必要でUX低下 |
