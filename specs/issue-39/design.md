# Issue #39: ダッシュボードのスマホ表示時のデザイン修正

## 問題分析

### 現状の動作
ダッシュボード（`/`）は月別サマリーテーブルを表示している。テーブルには13列（月、件数、男性、女性、会場費、見込み収入、決済済み、未回収、見込み利益、実現利益、見込み利益(CB込)、実現利益(CB込)、利益率）があり、スマホ画面（幅320px〜428px程度）では横スクロールが必要となる。

### 問題の原因

1. **多すぎるカラム数**
   - `src/components/dashboard/monthly-summary-table.tsx` (L30-L103)
   - 13列すべてが常に表示されるため、スマホでは横幅が足りない

2. **タイトルと年度セレクターの配置**
   - `src/app/page.tsx` (L21-L24)
   - `flex items-center justify-between` で横並び配置されているが、スマホでは狭くなる可能性

3. **セルのスタイル**
   - `src/components/ui/table.tsx` (L68-L91)
   - `whitespace-nowrap` により折り返しが防止され、金額表示が横幅を取る

4. **レイアウト全体**
   - `src/app/layout.tsx` (L21)
   - メインコンテンツ領域は `p-4 pt-16 md:p-6 md:pt-6` で適切なパディングあり

## 修正方針

### 変更対象ファイルと変更内容

#### 1. `src/components/dashboard/monthly-summary-table.tsx`
スマホ表示用のカード形式レイアウトを追加する。

- **変更内容**:
  - レスポンシブ対応: モバイル（<768px）ではカード形式、デスクトップではテーブル形式で表示
  - カード形式では重要な項目（月、件数、男女数、見込み利益、実現利益）を優先表示
  - 「詳細を見る」展開で全項目を表示可能に

#### 2. `src/app/page.tsx`
ヘッダー部分のレスポンシブ対応を強化する。

- **変更内容**:
  - タイトルと年度セレクターの配置を `flex-col sm:flex-row` に変更
  - スマホでは縦並び、タブレット以上では横並び

### 選択しなかった代替案

| 代替案 | 不採用の理由 |
|--------|--------------|
| 横スクロールのみ | 現状と変わらず、ユーザビリティが改善しない |
| 一部カラムの非表示 | 情報の欠落が発生し、業務に支障をきたす可能性 |
| フォントサイズを小さくする | 視認性が低下し、タップ操作も困難になる |
| 別ページに分割 | ダッシュボードの一覧性が失われる |

## 影響範囲

### 影響を受けるコンポーネント
- `MonthlySummaryTable` コンポーネント（直接変更）
- `DashboardPage`（直接変更）

### 影響を受けない部分
- データ取得ロジック（`dashboard-queries.ts`）
- 型定義（`MonthlySummaryRow`）
- ナビゲーション（既にレスポンシブ対応済み）

### 破壊的変更の有無
**なし** - 表示形式の変更のみで、データ構造やAPIに変更はない
